TRÍPTICO BIPOLAR — Long page de producción (tripticobipolar.es)
================================================================

QUÉ ES ESTO
-----------
Sitio estático autocontenido (un solo index.html con todo el CSS y el JS dentro),
listo para subir a Vercel. Sigue el Documento de Copywriting Maestro V3:
Hero → Fénix → Paradoja → Kala → Cierre del tríptico → Cómo funciona → FAQ →
Formulario de reserva → Footer.

----------------------

LO QUE TIENES QUE AÑADIR (2 minutos)
------------------------------------
1) PORTADAS — en assets/covers/ con estos nombres exactos:
     trio.png        (render de los 3 libros juntos — para el hero)
     fenix.png       (portada de Las plumas del Fénix)
     paradoja.png    (portada de Paradoja)
     kala.png        (portada de Kala)
   Si falta alguna, la web no se rompe: muestra un marco-placeholder en su lugar.

2) AUDIOS — en assets/audio/ con estos nombres exactos:
     fenix.mp3       (tu 003-LAS_PLUMAS-DEL_FENIX...)
     paradoja.mp3    (tu 004-PARADOJA_huir_hacia_delante)
     kala.mp3        (tu 005-KALA_la_deidad_de_la_destruccion)
   (No los pude copiar yo: pesan ~17 MB cada uno y el espacio de trabajo
    escribe muy lento. Arrástralos tú y renómbralos.)
   OJO: en assets/audio/ hay un "fenix.mp3" a medias (12 MB, copia
   incompleta que se cortó). BÓRRALO y pon el tuyo completo encima.
   Los audios 001 y 002 (generales) no se usan en la página; guárdalos por si
   quieres un episodio "intro" más adelante.

SUBIR A VERCEL
--------------
Opción A (sin instalar nada):
   1. Entra en vercel.com → New Project → Deploy.
   2. Arrastra la carpeta web-tripticobipolar entera (o súbela a un repo de
      GitHub y conéctalo).
   3. Framework Preset: "Other". Output: la propia carpeta (sin build).
   4. Deploy. Luego añade tu dominio tripticobipolar.es en Settings → Domains.

Opción B (terminal):
   npm i -g vercel
   cd web-tripticobipolar
   vercel        (sigue las preguntas; "vercel --prod" para producción)

NOTA SOBRE EL FORMULARIO
------------------------
El formulario es un MVP sin backend: al enviar muestra en pantalla las
instrucciones de pago (Bizum / transferencia) con el nombre del cliente.
NO guarda el pedido en ningún sitio ni te avisa. Para recibir los pedidos por
email, conéctalo a un servicio gratuito (Formspree, Vercel Forms o similar):
   - Cambia <form id="reservaForm"> por la action del servicio, o
   - dímelo y te lo dejo cableado.
Recuerda rellenar los datos reales de pago (número de Bizum e IBAN) — ahora
están como XXXX en el JavaScript del final del index.html.

PENDIENTE / RECOMENDADO
-----------------------
- Testimonios reales de lectores (bloque de prueba social) — no incluido para
  no inventar reseñas; lo añadimos cuando tengas citas verificables.
- Cluster de keywords (Nuria) para afinar los H2/H3 y meter long-tail.
- Color: la web usa el fucsia de marca #E0067A. El V3 mencionaba #FF0066;
  si prefieres ese, cámbialo en la variable --fucsia (arriba del index.html).
- Páginas legales (Aviso legal / Privacidad / Condiciones) — los enlaces del
  footer apuntan a # de momento.
